# 🏭 IoT Sensor Monitoring Data Platform v1.0.0

## 🎉 Project Launch

Complete **end-to-end data platform** for real-time IoT sensor monitoring in manufacturing environments. This release delivers a production-ready solution with enterprise-grade microservices architecture and automated data processing.

## 🚀 Core Features

### 📊 **Data Engineering Pipeline**

- ⚡ **Apache Airflow 2.10.3** - Automated ETL with hourly scheduling
- 🗄️ **PostgreSQL + TimescaleDB** - Time-series data optimization  
- 🔄 **Real-time Ingestion** - IoT temperature/humidity sensor data
- 📈 **Aggregation Engine** - Time & location-based analytics

### ☸️ **Kubernetes Microservices (9 Services)**

- 🎯 **FastAPI + SQLAlchemy 2.0** - RESTful API architecture
- 🔧 **Thresholds Management** (Port 30001) - Dynamic threshold configuration
- 📍 **Location Service** (Port 30002) - Sensor location mapping
- ⚡ **Realtime Service** (Port 30003) - Live data & threshold monitoring
- 📊 **Aggregation Service** (Port 30004) - Statistical processing
- 🚨 **Alert System** (Ports 30007-30009) - Multi-channel notifications
- 📚 **Integrated Swagger** (Port 30005) - Unified API documentation

### 🛡️ **Enterprise Features**

- 🚨 **Multi-Channel Alerting**: Email, KakaoTalk, SMS, Mobile App
- 📈 **Auto-Scaling**: Kubernetes HPA for dynamic resource management
- 🔍 **Monitoring**: Prometheus integration for observability
- 🐳 **Containerization**: Full Docker + Kubernetes deployment

## 🏗️ **Technical Architecture**

- **Pipeline**: Hourly ETL with validation & retry logic
- **Infrastructure**: Kind-based Kubernetes with service mesh
- **Data Flow**: Real-time ingestion → Processing → Aggregation → Alerting

### 🔧 **Infrastructure Components**

✅ **Manufacturing IoT Monitoring** - Temperature/humidity tracking

## 🚀 **Getting Started**

1. **Data Pipeline**: Deploy Airflow with `docker-compose up`
2. **Microservices**: Deploy to Kubernetes cluster  
3. **Monitoring**: Access unified APIs at port 30005
4. **Documentation**: Each service includes detailed README
